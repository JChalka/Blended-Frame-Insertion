# Cube to Header Converter

Convert standard `.cube` 3D LUT files into formats compatible with the TemporalBFI `CubeLUT3D` loader.

## Supported Input

- **Iridas / Resolve `.cube` files** — the de-facto standard produced by DisplayCAL, DaVinci Resolve, ArgyllCMS, CalMAN, and most display profiling tools.
- RGB (3-channel) and RGBW (4-channel, non-standard) cubes.
- `LUT_3D_SIZE`, `DOMAIN_MIN`, `DOMAIN_MAX`, and `TITLE` keywords are parsed; unknown keywords are skipped.

## Output Formats

| Format | Extension | Description |
|--------|-----------|-------------|
| Binary | `.bin` | 4-byte header (grid uint16 LE, channels uint16 LE) followed by interleaved Q16 payload. Directly loadable via `CubeLUT3D::loadFromFileBuffer()`. |
| C++ Header | `.h` | PROGMEM `uint16_t` array with interleaved RGBW/RGB data, plus a `FILE_HEADER[4]` constant. Include in your sketch and copy from flash via `memcpy_P` before calling `CubeLUT3D::attach()`. |

## Usage

```
python cube_to_header.py <input.cube> [options]
```

### Arguments

| Argument | Description |
|----------|-------------|
| `input` | Path to the `.cube` file to convert (required) |
| `-o`, `--output` | Output base path without extension. Defaults to the input file stem. Produces `<base>.bin` and/or `<base>.h`. |
| `--name` | C identifier prefix for the header arrays (default: derived from output filename, uppercased). Example: `--name MY_DISPLAY_LUT` produces `MY_DISPLAY_LUT_DATA[]`, `MY_DISPLAY_LUT_GRID_SIZE`, etc. |
| `--binary-only` | Write only the `.bin` file |
| `--header-only` | Write only the `.h` file |

### Examples

```bash
# Convert a DisplayCAL profile to both binary and header
python cube_to_header.py my_display.cube

# Header only, custom prefix
python cube_to_header.py my_display.cube --header-only --name DISPLAYCAL_LUT

# Binary only, custom output path
python cube_to_header.py calibration.cube -o output/cal_cube --binary-only
```

## Loading on the MCU

### From SD / PSRAM (binary)

```cpp
// Read the .bin file into a byte buffer, then:
CubeLUT3D cube;
uint16_t grid; uint8_t ch;
CubeLUT3D::parseHeader(fileBuffer, grid, ch);
uint16_t* buf = (uint16_t*)extmem_malloc(CubeLUT3D::dataBytes(grid, ch));
cube.data = buf;
cube.loadFromFileBuffer(fileBuffer, fileSize);
solver.setCubeLUT3D(&cube);
```

### From PROGMEM header

```cpp
#include "my_display.h"  // generated by cube_to_header.py

CubeLUT3D cube;
uint16_t* buf = (uint16_t*)malloc(MY_DISPLAY_DATA_LEN * sizeof(uint16_t));
memcpy_P(buf, MY_DISPLAY_DATA, MY_DISPLAY_DATA_LEN * sizeof(uint16_t));
cube.attach(buf, MY_DISPLAY_GRID_SIZE, MY_DISPLAY_CHANNELS);
solver.setCubeLUT3D(&cube);
```

## Data Layout

Values are Q16 (0–65535), scaled from the `.cube` file's 0.0–1.0 float range. Data is stored in R-major order with interleaved channels:

```
index = (r * N * N + g * N + b) * channels + ch
```

This matches the `CubeLUT3D` struct's expected memory layout exactly.
